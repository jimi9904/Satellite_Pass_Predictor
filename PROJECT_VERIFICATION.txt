"This is a Real-Time ISRO Satellite Pass Predictor application. It allows users to predict when ISRO satellites will pass over their location, view real-time satellite positions on a map, and learn about ISRO missions."

POTENTIAL VIVA QUESTIONS AND ANSWERS
================================================================================

Q: Why did you separate frontend and backend?
A: Separation allows independent development, deployment, and scaling. Frontend can be deployed to CDN (Vercel), backend to cloud server (Render/Railway). They communicate via HTTP API, so they can be on different servers.

Q: How does authentication work?
A: User logs in with email/password. Backend verifies credentials, hashes password with bcrypt, and returns JWT token. Frontend stores token in localStorage and includes it in Authorization header for all API requests. Backend middleware verifies token on protected routes.

Q: How do you calculate satellite positions?
A: We use TLE (Two-Line Element) data stored in MongoDB. The tle.js library calculates current position from TLE data. We also calculate orbit paths by sampling positions over one orbit period. Positions are updated in real-time every 5 seconds.

Q: Why MongoDB instead of SQL?
A: MongoDB's flexible schema fits satellite data well. TLE data is just two strings, no complex relations needed. Easy to add new satellite fields without migrations. Good performance for read-heavy workload.

Q: How do you ensure TLE data is current?
A: Background job runs daily at 2 AM IST using node-cron. It fetches fresh TLE data from CelesTrak API and updates all satellites in database. Also runs immediately on server startup.

Q: What happens if backend is down?
A: Frontend still loads but API calls fail. Error messages are displayed to users. Frontend gracefully handles errors and shows appropriate messages.

Q: Can frontend work without backend?
A: Frontend can start and display UI, but all features require backend API. Login, satellite data, predictions, and map all need backend. Frontend is a client that depends on backend services.

Q: How do you handle errors?
A: Backend returns appropriate HTTP status codes (400, 401, 404, 500) with error messages. Frontend catches errors in try-catch blocks and displays user-friendly messages. Network errors are handled by Axios interceptors.

Q: What security measures are implemented?
A: Passwords hashed with bcrypt, JWT tokens for authentication, CORS protection, input validation, environment variables for secrets, no SQL injection (using Mongoose), no XSS (backend only returns data).

Q: How does the map update in real-time?
A: MapView component uses setInterval to call /api/position/:satId every 5 seconds. Backend calculates current position from TLE data. Frontend updates map marker and orbit path. Backend caches position for 5 seconds to prevent redundant calculations.
